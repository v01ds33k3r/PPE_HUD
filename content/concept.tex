\subsubsection{Visualizing Oxygen Usage Through a Status Bar}

Since the oxygen value is the crucial factor whether or not to continue work, this is the component taking the spotlight. As the most critical parameter, this component will be visible in any version of the HUD. To check the current status as quickly as possible, the value will be visualized with a status bar.

Since mechanical gauges with a pointer show visual progress by default, a radial version of the component was included.
% Additionally, the component takes up grid space shaped more as a square, which offers different layout options. 
However, as mentioned in Section \ref{sec:datavis}, linear design enable much quicker parsing of the information. Compared to the linear version, it also takes up much more space to make the information readable.

\begin{figure}[hbt]
    \centering
    \includegraphics[width=0.8\textwidth]{assets/oxy_radial_bar.eps}
    \captionbelow{The initial concepts for the status bar, considering overall shape.}
    \label{fig:oxy_radial_bar}
\end{figure}

The second version places the value below the bar, on the right. When the amount of oxygen decreases, the empty portions of the bar are only a contour. Thus, the contrast between value and bar becomes stronger. 
The label is placed in front of the bar so they blend together, making it less salient.
Additionally, the status bar was divided into quarter segments, enabling more accurate estimation of the amount left, without looking at the value.

Depending on available screen space and layout configuration, a vertical version of the component might be needed in addition to the horizontal one. 

Grouping the label and value on one side saves space (left and middle of Figure \ref{fig:oxy_vertical}). Aligning them at the top and bottom respectively accounts for visual crowding, making the value more distinctive.
The oxygen value is located so that the status bar is seen first, i.e. if the component is placed on the left side, the bar will be on the right to maintain an optical guide.

\subsubsection{The Base Component}

This element is built with a simple configuration for secondary information that is helpful, but does not need to be prioritized. This includes data like heart rate, but also environmental information such as ambient temperature or humidity.

Values that are not to be visualized are similar enough to be based on a universal base component. Apart from the actual parameter value, any of these elements can be hidden. This allows flexible configuration depending on the type of information.

\begin{figure}[hbt]
    \centering
    \includegraphics[width=0.6\textwidth]{assets/base_component.eps}
    \captionbelow[The base component]{The base component with configurable icon, parameter value, unit and trend indicator.}
    \label{fig:base_component}
\end{figure}

The icons replace labels, and should therefore be as unambiguous as possible.
% If the icon is unambiguous enough, it is easier to hide the unit. 
Leaving the unit out saves space, though in some cases, it supports the identification of the value (e.g., degrees Fahrenheit or Celsius).

The trend indicators are displayed based on the difference of the current and previous value. The value increasing or decreasing an unusual amount might indicate events to take notice of. For dynamic elements like these, the frequency of changes should not be too high to avoid distraction. Comparing the short and long arrows, the long arrows are better readable when out of focus.

\begin{figure}[hbt]
    \centering
    \includegraphics[width=0.8\textwidth]{assets/arrow_shape.eps}
    \captionbelow[Comparison of short and long arrows]{Comparison of short and long arrows.}
    \label{fig:arrow_shape}
\end{figure}

All of these values are displayed continuously, with the exception of the time component. It will only be displayed after a configured amount of time. The element popping up is meant to be perceivable in the peripheral vision, marking (for example) the halfway point, encoding this action with information.
Since working time is limited by various parameters and therefore not fixed, the component is formatted as a countdown. To avoid pulling unnecessary attention, the display is limited to minutes, rather than ticking down by the second.

Referencing typical industrial warning signs, the hazard icons were created in a triangular and a diamond-shaped version. Comparing them however, reveals that for the same size, the diamond shape appears smaller. This also means that there is less space for the graphic inside it.
Avoiding unique silhouettes unites them in their function.  For peripheral distinction, their placement is therefore more important.
The icons are placed close together so that they are visible at the same angle if multiple hazards are present. Similar to a car's dashboard warning lights, each of them has a fixed position, facilitating rapid information acquisition. With regular notifications—depending on which one appears first—the order of content would be different.

\subsubsection{The Checklist Component}

% combines interaction paradigms introduced by \cite{luGlanceableAREvaluating2020}

Since this element is text-based, it relies less on peripheral vision.
To anchor the gaze to the current task, it is centered vertically and more distinctive than the other displayed tasks.
%Though the other tasks are smaller, everything is left-aligned to—once again—create an optical guide.

%sentence case vs lower case

Note that this component is meant as a visual example, which is why certain aspects of functionality might not work in regard to handling the list. (e.g. if only two next tasks are displayed, there is no way to see if new tasks got added -> task number display in dynamic situations, generally better for protocol).

\begin{figure}[hbt]
    \centering
    \setlength{\fboxsep}{24pt}
    \setlength{\fboxrule}{2pt}
    \fbox{\includegraphics[width=0.7\textwidth]{assets/checklist.eps}}
    \captionbelow[Checklist Component]{Default and expanded checklist. Note the center alignment of the current task to avoid losing focus when the list expands.}
    \label{fig:checklist}
\end{figure}

The icon to the right is an anchor for possible gaze-based interaction, in this case to show more details. This design keeps the current task centered in the detailed view as well for faster orientation. This also mitigates side effects from possible false positives—i.e. if the system mistakenly opens the detailed view.
