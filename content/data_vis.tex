\section{Data Visualization}
\label{sec:datavis}

"One of the greatest benefits of data visualization is the sheer quantity of information that can be rapidly interpreted if it is presented well" [\cite{wareChapterOneFoundations2013}].

Data visualization transforms data into information. Data itself is just noise, whereas information holds value—value that is created by the context through which the data is interpreted.

\subsection{Guiding Visual Search}

Information needs to be located before it can be interpreted. To make this happen as quickly as possible, certain design principles can be utilized.

To this end, \textcite{shaoHowDynamicInformation2023} present the 3V method, which refers to visual hierarchy, weight, and flow respectively.
The visual weight is described by the features of and space occupied by elements. It works through the distinction of objects, achieved by differing presentation. More visual weight means the object attracts more attention.
The visual connections between elements describe the visual flow. If implemented correctly, flow guides the gaze to target information.

In summary, even when a user is consciously (i.e., top-down) searching for a specific element in the user interface, a well-structured design can leverage bottom-up attention to guide this visual search [\cite{shaoHowDynamicInformation2023}].

\subsection{Facilitating Rapid Comprehension}

Despite the majority of data visualization being tailored to complex datasets, some concepts can still be utilized to make information glanceable.

\textcite{wilke2VisualizingData2019} identifies position, shape, size and color as the fundamental elements for visualization. Position describes the arrangement of data in relation to other points or axes in a diagram. As for color, a distinction should be made between hue and brightness, as these are appropriate for different types of data. Hues are better suited for clearly defined categories, while brightness is best for continuous data, since the eye is more sensitive to differences in brightness and can therefore distinguish smaller variations.

Each of these elements can be used to encode information. Which element is used for which information influences the type of visualization. Importantly, this mapping must be unique, else it becomes ambiguous and obscures the meaning [\cite{wilke2VisualizingData2019}].

\subsection{Visualizations for Single Value Data}

Aside from design fundamentals, most specialized resources discuss visualization types intended for interrelated data. Interpreting such complex information requires time and cognitive capacity unavailable in scenarios such as those mentioned in Section \ref{sec:CPC}. In these situations, a clear picture of the current status must be available at all times, which only allows for individual data values and therefore limits the number of possible visualization types.

Ferdio, an agency specialized on infographics and data visualization, has created a library of visualization types\footnote{\href{https://datavizproject.com/}{Data Viz Project | Collection of data visualizations to get inspired and find the right type}}. In this library, out of a 166 options, only nine are for single data (see Figure \ref{fig:ferdio_single_data}).
